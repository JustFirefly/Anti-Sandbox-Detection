1.0 EBPF KERNEL & USER-SPACE INSTRUMENTATION
-----------------------------------
Objective: Evaluate eBPF capabilities for modifying system and application return values across kernel, user-space, and virtual machine boundaries.
1.1 Return Value Modification
    - Investigated `bpf_override_return()` mechanisms and identified its strict limitation to allow-listed kernel functions (kprobes).
    - Analyzed the constraints of modifying user-space functions (uprobes/uretprobes) and the inherent risks of stack manipulation via `bpf_probe_write_user()`.
1.2 Virtual Machine Interactions
    - Mapped the Host-to-VM boundary, differentiating interception strategies for System VMs (QEMU/KVM `ioctl` interception) versus Language VMs (JIT compilation / USDT probes).
    - Established that hardware-level instructions like `RDTSC` cannot be natively hooked via standard eBPF without inducing highly detectable hypervisor VM-exit latency.

2.0 SANDBOX DETECTION & EVASION STRATEGY
-----------------------------------
Objective: Analyze a custom Windows C++ sandbox detection payload and formulate host-level bypass methodologies.
2.1 Payload Analysis (T01-T08)
    - Evaluated an 8-stage detection PoC encompassing CPUID hypervisor bits, MAC addresses, specific driver files, analysis processes, and RDTSC timing heuristics.
    - Determined that Linux-based `LD_PRELOAD` is ineffective against inline assembly instructions (e.g., MSVC `__rdtsc()`) due to the absence of dynamic linker symbol resolution.
2.2 Evasion Methodology Formulation
    - Outlined environment-level bypasses (e.g., hypervisor configuration) versus technical bypasses (e.g., API hooking via MinHook/Detours, binary patching).
    - Investigated `VBoxManage` configuration techniques for neutralizing hardware and network detection vectors specifically for VirtualBox on Ubuntu hosts.

3.0 DYNAMIC HYPERVISOR INSTRUMENTATION (LINUX HOST)
--------------------------------------
Objective: Develop a functional payload to intercept and modify QEMU/KVM hypervisor initialization parameters from user-space.
3.1 KVM ioctl Hooking (kvm_hook.so)
    - Engineered a C-based shared library to hook standard `ioctl()` calls executed by the QEMU process.
    - Implemented payload logic to successfully intercept the `KVM_SET_CPUID2` request from QEMU to the host KVM module.
    - Configured the hook to dynamically clear the "Hypervisor Present" bit (ECX[31] on Function 1) and wipe hypervisor vendor strings (Function 0x40000000) prior to VM execution.

4.0 VIRT-MANAGER INTEGRATION & SECURITY REMEDIATION
---------------------------------------
Objective: Inject the custom `LD_PRELOAD` payload into a libvirt-managed QEMU virtual machine and systematically resolve OS-level security blocks.
4.1 Libvirt XML & Process Modification
    - Attempted direct `LD_PRELOAD` injection via the `<qemu:commandline>` namespace in the `virsh` XML configuration.
    - Diagnosed "silent failures" (missing standard output and missing environment variables) caused by the libvirtd daemon's aggressive environment sanitization.
4.2 Wrapper Script Implementation
    - Developed `/opt/qemu-wrapper.sh` to act as an execution middleman, explicitly forcing the `LD_PRELOAD` export prior to executing the actual QEMU emulator binary.
    - Reconfigured the target VM's `<emulator>` tag to point to the custom wrapper script.
4.3 AppArmor/SELinux Remediation
    - Resolved `virsh edit` XML rejection errors by updating the local AppArmor `usr.sbin.libvirtd` profile to permit capabilities probing on the wrapper script.
    - Remediated fatal "Permission denied" execution errors during VM boot by appending the wrapper script and `.so` payload paths to the global `libvirt-qemu` AppArmor abstraction, successfully granting the restricted guest process permission to load the hook.
