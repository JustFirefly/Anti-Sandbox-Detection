1.0 HOST & HYPERVISOR SPECIFICATION
-----------------------------------
Objective: Establish a stable virtualization platform capable of supporting 
Windows 11 guest execution and kernel-level modification, per PDD Risk R3 
(Test Environment Instability).

1.1 Hypervisor Selection
    - Software: Oracle VirtualBox (Version 7.0.14 r161095 (Qt5.15.2)).
    - Justification: Selected as the "stable, mainstream hypervisor" outlined in 
      the Risk Register to minimize environment instability and support rapid 
      snapshot recovery.

1.2 Host Configuration
    - Processor: Intel with VT-x enabled (Nested Paging active).
    - Memory: 4GB RAM allocated to Guest (ensures responsiveness during runtime).
    - TPM Configuration: Enabled TPM 2.0 Passthrough/Emulation to satisfy Windows 11 installation checks without modifying the integrity of the Guest OS image.

2.0 GUEST OPERATING SYSTEM CONFIGURATION
----------------------------------------
Objective: Deploy a target operating system consistent with the Work Plan 
(Phase 2.2) for cataloguing Windows-based detection techniques.

2.1 OS Version
    - System: Windows 11 Enterprise (Evaluation Edition).
    - Source: Official Microsoft ISO.

2.2 Initial Setup State (The "Bait")
    - VirtualBox Guest Additions: INSTALLED.
      * Reasoning: Essential for the "Pre-Spoof Test" (Objective 3). The presence 
        of VBoxGuest.sys, registry keys, and specific file paths creates the 
        standard artefacts required for detection algorithms to successfully 
        identify the sandbox.

2.3 Security Controls
    - Windows Defender/SmartScreen: DISABLED.
      * Reasoning: Prevents interference with non-malicious Proof-of-Concept (PoC) 
        tools. The project focus is sandbox environment detection, not Anti-Virus evasion mechanisms.

3.0 DEVELOPMENT WORKFLOW & ISOLATION
------------------------------------
Objective: Mitigate Risk R3 by separating development from execution.

3.1 Host Role (Development)
    - Tools: Visual Studio 2022, Windows SDK/WDK.
    - Activity: Writing and compiling C/C++ detection/evasion tools.
    - Storage: All source code maintained in private repository on Host.

3.2 Guest Role (Execution)
    - Tools: Visual C++ Redistributable (Debug/Release) installed.
    - Activity: Execution of compiled binaries only.
    - Data Transfer: Read-Only Shared Folder configured to transfer .exe PoCs 
      from Host to Guest.

4.0 SNAPSHOT MANAGEMENT STRATEGY
--------------------------------
Objective: Ensure experimental repeatability and rapid recovery from potential 
kernel-level crashes.

4.1 Snapshot Hierarchy
    - Snapshot A: "Base Install"
      * State: Clean Windows 11 installation. No drivers, no updates.
    
    - Snapshot B: "Golden Image (Vulnerable)"
      * State: Guest Additions installed, VC++ Runtimes installed, Defender disabled.
      * Usage: This is the primary rollback point. It ensures the "Pre-spoof test" 
        (detection success) works immediately.

    - Snapshot C: "PoC Staging"
      * State: Environment modified with evasion PoCs (e.g., API hooks).
      * Usage: Used for "Post-spoof test" verification.
