### Why `LD_PRELOAD` Fails Here

`LD_PRELOAD` works by intercepting **library function calls** (dynamic symbols) like `printf`, `malloc`, or `open`.

In your `t06_rdtsc_check()` function, you are using the MSVC intrinsic `__rdtsc()`:

```cpp
unsigned long long rdtsc_diff() {
    unsigned long long tsc1, tsc2;
    tsc1 = __rdtsc(); // This is an INLINE assembly instruction
    tsc2 = __rdtsc(); 
    return tsc2 - tsc1;
}

```

When you compile this code, the compiler does not generate a call to an external library. Instead, it embeds the raw machine code for the `RDTSC` instruction directly into your executable's `.text` section.

Since there is no "call" to a library, there is no symbol for `LD_PRELOAD` to "hook" or "redirect." The code bypasses the dynamic linker entirely.
